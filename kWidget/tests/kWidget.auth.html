vWidget Authentication

vWidget Authentication enables onPage applications to trigger a remote login, and retrieve a Vidiun secret, for performing admin actions. This enables javascript applications, to authenticate with Vidiun without exposing user login credentials to the domain or having users login in on non-secure entry points.

Display a authentication status & login link, with vs callback


The easiest way to authenticate is with vWidget.auth.getWidget, which will:
  • Provide a branded login link in the hosted target
  • Issue a callback with authentication credentials, as soon as the user logins
  • Once the user is already logged in on vidiun.com, and has approved the application domain, it will issue the callback directly in future page loads 
<script src="http//cdnapi.vidiun.com/p/{partnerId}/sp/{partnerId}00/embedIframeJs/uiconf_id/{uiconf_id}/partner_id/{partnerId}"></script>
<script src="http://player.vidiun.com/vWidget/vWidget.auth.js"></script>
<div id="hostedAuthWidget"></div>
 <script> 
	vWidget.auth.getWidget( "hostedAuthWidget", function( userObject ){
		$authTable = $('<table>').css( 'width', '350px' );
		$.each( userObject, function( key, value){
			$authTable.append( 
				$('<tr>').append(
					$('<td>').text( key ),
					$('<td>').text( value )
				)
			)
		})
		$('#hostedAuthWidget').after( 
			"Got login info: ",
			$authTable 
		)
	})
</script>
